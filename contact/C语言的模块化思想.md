# C语言的模块化思想

## 一个模块的封装

```c
//lcd.c
void lcd_init(void)
{
    xxxxxxxx;
}

//lcd.h
void lcd_init(void);
```

```c
#include "lcd.h"
lcd_init();
```

如上，c语言中一个模块一般对应一个C文件和头文件，模块的实现在C源文件中，头文件主要用来存放函数声明，既模块的API声明

需要调用此API只需要引入这个头文件即可使用

## 防止头文件被重复包含的方法

当头文件被重复包含展开时则编译时可能会报重定义错误，为了防止这种错误产生，可以在头文件中使用条件编译来预防头文件多次包含

```c
//lcd.h
#ifndef __LCD_H__
#define __LCD_H__
...
#endif
```

这种方法可以保证无论包含了几次，预处理过程只展开一次

## 隐式声明

当使用一个未经声明的函数时，编译器会隐式地对其做一个声明，并且报一个警告（C99），在C（11）时则会报错，但是在C99时会通过编译导致不可控的错误，因此对于这种警告要注意！

> warning: implicit declaration of function 'xxx'
